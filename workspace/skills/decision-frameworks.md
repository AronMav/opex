## Decision-Making Frameworks

Use a structured approach instead of intuitive choice.

### When to apply

- There are 2+ options and it's unclear which is better
- The decision has significant consequences
- User asks to compare or choose

### Framework 1: Quick Assessment (Pros/Cons+)

For simple decisions with 2-3 options:

```
| Criterion       | Option A  | Option B  |
|-----------------|-----------|-----------|
| Pros            | ...       | ...       |
| Cons            | ...       | ...       |
| Risks           | ...       | ...       |
| Reversible?     | yes/no    | yes/no    |
| → Recommendation| ...                   |
```

**Rule**: if the decision is reversible — choose faster. If irreversible — analyze more deeply.

### Framework 2: RICE (for task prioritization)

```
RICE Score = (Reach × Impact × Confidence) / Effort

Reach:      how many people/processes affected (1-10)
Impact:     how strongly it will impact (0.25 / 0.5 / 1 / 2 / 3)
Confidence: confidence in estimates (0.5 / 0.8 / 1.0)
Effort:     work in person-days
```

### Framework 3: Decision Matrix (for complex choices)

```
| Criterion (weight)   | Option A  | Option B  | Option C  |
|----------------------|-----------|-----------|-----------|
| Cost (0.3)           | 8 → 2.4   | 5 → 1.5   | 3 → 0.9   |
| Speed (0.25)         | 6 → 1.5   | 8 → 2.0   | 9 → 2.25  |
| Quality (0.25)       | 9 → 2.25  | 6 → 1.5   | 4 → 1.0   |
| Risk (0.2)           | 7 → 1.4   | 5 → 1.0   | 8 → 1.6   |
| TOTAL                | 7.55      | 6.0       | 5.75      |
```

### Framework 4: Second-Order Effects

For strategic decisions — think one step ahead:
1. If we choose A — what happens in a week? Month? Year?
2. What new opportunities will open up?
3. What opportunities will be closed off?
4. What is the worst that could happen?

### Principles

- Don't analyze longer than the decision is worth
- For everyday decisions — Pros/Cons in 30 seconds
- For important ones — RICE or matrix
- For strategic ones — second-order effects
- Always name a recommendation, never leave a choice without an answer
