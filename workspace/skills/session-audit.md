# Session Audit — Аудит Сессий

## Назначение

Систематический анализ истории диалогов: поиск, восстановление контекста, выявление паттернов, ошибок и аномалий.

## Процедура

### Шаг 1 — Классификация запроса

Определи тип аудита:

| Тип | Признак | Действие |
|-----|---------|----------|
| **Точечный поиск** | «что было в сессии X», «найди где обсуждали Y» | `session(action="search")` → выдать результат |
| **Обзорный** | «покажи последние сессии», «что сегодня было» | `session(action="list")` → сводка |
| **Глубокий аудит** | «проанализируй сессии за период», «найди все ошибки» | Полный цикл ниже |
| **Восстановление** | «потерял контекст», «напомни о чём говорили» | `session(action="history")` → контекст |

### Шаг 2 — Сбор данных (глубокий аудит)

```
1. session(action="list", limit=50) → список сессий
2. Для каждой релевантной: session(action="history", session_id="...")
3. Параллельно: memory(action="search", query="...") — контекст из памяти
```

### Шаг 3 — Анализ

Проверяй сессии на:

- **Незавершённые задачи** — были запросы без подтверждения выполнения
- **Ошибки инструментов** — неудачные вызовы, таймауты, отказы
- **Дублирование** — повторные запросы одной и той же сути
- **Конфликты** — противоречивые ответы в разных сессиях
- **Потерянный контекст** — темы, которые оборвались без завершения
- **Паттерны** — часто повторяющиеся запросы, типовые проблемы

### Шаг 4 — Отчёт

Формат для глубокого аудита:

```
## Аудит сессий: [период/тема]

### Общая сводка
- Всего сессий: N
- Проанализировано: N
- Найдено аномалий: N

### Найденное

#### Незавершённые задачи
- [сессия] [описание]

#### Ошибки
- [сессия] [инструмент] [ошибка]

#### Паттерны
- [паттерн] [частота]

### Рекомендации
- [действие]
```

### Шаг 5 — Фиксация (опционально)

При выявлении важных находок:
```
memory(action="index", content="[находка]", source="session-audit")
```

## Правила

1. **Не перечитывай свои файлы.** SOUL.md, IDENTITY.md и прочие загружаются автоматически.
2. **Сначала список, потом содержимое.** Не лезь в историю конкретной сессии без явной необходимости.
3. **Контекст текущего канала учитывай.** Если канал — agent (внутренний), отчёт должен быть плотным. Если telegram — адаптируй под сообщение.
4. **Не дублируй.** Если аудит уже выполнялся недавно — сошлись на предыдущий, не повторяй.
5. **Экономия токенов.** При большом количестве сессий делай выборку: последние N дней, конкретный агент, конкретная тема.
