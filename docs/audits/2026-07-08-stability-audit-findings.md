### F001 — [S1/CONFIRMED] The <think>-block stripping loop in parse_extraction (and the identical loop in update_rolling_summary) spins forever and grows the string unbounded whenever a '</think>' token occurs textually before the first '<think>'.

- **Location:** `crates/opex-core/src/agent/knowledge_extractor.rs:271`  ·  **axis 1** (crash/panics)  ·  **layer** rust-core  ·  **cell** `core-memory`  ·  **blast** whole-system  ·  **score** 323
- **Failure scenario:** extract_and_save runs in a spawned background task (finalize.rs spawn_knowledge_extraction). It calls parse_extraction on the extraction LLM's raw content, then update_rolling_summary on the summary LLM's content. The loop does `while let Some(start) = content.find("<think>")` and, inside, `if let Some(end) = content.find("</think>")` — but find("</think>") returns the FIRST closing tag regardless of position. If content is e.g. `</think><think>` (byte end=0, start=8), the rebuild `&content[..start] + &content[end+8..]` = `</think>` + `<think>` = the SAME string, so `<think>` is never removed and the loop never terminates; when end+8 < start the rebuilt string is strictly longer each iteration, growing without bound. The content is user-influenceable: a user pastes `</think><think>` into a conversation (or a reasoning model emits a stray/leading closing tag), the summarized text is echoed by the LLM, and the background task pegs a CPU core forever and/or exhausts RAM until the whole opex-core process is OOM-killed. Existing tests only cover well-ordered / unclosed tags, not the reversed-order case; the `// reviewed: ... char boundaries` comment shows the byte-offset safety was reviewed but the ordering bug was missed.
- **Evidence:** knowledge_extractor.rs:270-277 (parse_extraction) and :238-245 (update_rolling_summary) both use `content.find("</think>")` (unanchored) inside `while content.find("<think>")`. finalize.rs:701 spawns extract_and_save as a background task, so the hang/OOM is unattended.
- **Proposed fix:** Search for the closing tag only after `start`: `let rest = &s[start..]; if let Some(rel) = rest.find("</think>") { let end = start + rel; s = format!("{}{}", &s[..start], &s[end+8..]); } else { s.truncate(start); break; }`. Apply to both sites.
