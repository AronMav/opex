# Voice UX Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Починить голосовой web-чат: чистый транскрипт (без тайм-кодов), голос во время стриминга → очередь (не interrupt), стриминговая озвучка ответа по предложениям, защита от `(Empty response:`-мусора, и восстановить гейтинг голосовых кнопок на agent-capabilities (после профилей `provider_active` стал embedding-only → микрофон сейчас скрыт).

**Architecture:** Три слоя. toolgate: `/transcribe` снимает тайм-коды общим хелпером. core: нормализация upstream-мусора в `execute.rs` + включение empty-retry на интерактивном пути. UI: новый провайдер-агностичный модуль `tts-speaker.ts` (SentenceSplitter + SpeakerQueue) заменяет пост-фактум озвучку; голосовая отправка во время хода уходит в очередь; голосовые кнопки гейтятся на `AgentInfo.capabilities`.

**Tech Stack:** Python/FastAPI (toolgate), Rust (opex-core), Next.js 16 + React 19 + Zustand + vitest (UI).

**Spec:** [docs/superpowers/specs/2026-07-15-voice-ux-redesign-design.md](../specs/2026-07-15-voice-ux-redesign-design.md)

**Interfaces:**
- Produces:
  - `export function createSentenceSplitter(opts?: { minLen?: number }): { push(delta: string): string[]; flush(): string[] }` — `push` возвращает готовые предложения (граница `[.!?…]`+пробел/EOL, закрывающие кавычки/скобки прилипают, min длина 20), `flush` отдаёт остаток. Перед выдачей — markdown-очистка (заголовки/списки→плоско, код-блоки→«(код)», ссылки→текст).
  - `export interface SpeakerDeps { synth(sentence: string, signal: AbortSignal): Promise<Blob | null>; play(blob: Blob): Promise<void>; onStateChange?(s: "idle" | "speaking"): void; onDrain?(): void }`
  - `export function createSpeakerQueue(deps: SpeakerDeps): { enqueue(sentence: string): void; takeoverAudio(blob: Blob): void; cancel(): void; readonly idle: boolean }` — воспроизведение по порядку на едином элементе (через `deps.play`), prefetch=1 (синтез следующего параллельно проигрыванию текущего), `takeoverAudio` очищает очередь + abort'ит синтезы + играет переданный blob, `cancel` abort'ит всё + сбрасывает.

DOM-free ядро: fetch/audio инъектируются через `deps` (как `vad.ts` DOM-free). Это делает модуль юнит-тестируемым без jsdom-аудио.
