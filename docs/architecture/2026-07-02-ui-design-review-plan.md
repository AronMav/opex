# Дизайн-ревью Web UI — план правок

**Дата:** 2026-07-02
**Область:** `ui/` (Next.js 16, Tailwind 4, shadcn-примитивы, ~24 страницы, RU/EN, светлая/тёмная темы)
**Метод:** многоагентное ревью кода по 11 направлениям (дизайн-токены, страницы по группам, чат, workspace, входные страницы, a11y, адаптивность, копирайтинг/i18n) с адверсариальной верификацией каждой находки двумя независимыми проверками + визуальный осмотр живого UI (десктоп 1440px / мобильный 390px, обе темы, скриншоты).
**Итог:** 77 подтверждённых проблем (medium/high), 45 мелких (low), 2 находки отклонены при верификации.

> **Статус (2026-07-02):** первая итерация выполнена (~60–65%: этапы 1 и 5 — полностью, 2–3 — большей частью). Верификация и план доработки оставшегося — [фаза 2](2026-07-02-ui-design-review-phase2-plan.md).
>
> **Статус (2026-07-03): закрыто.** Фаза 2 выполнена и верифицирована на живом UI (10 критериев приёмки); error-banner дочинен отдельным фиксом (`extractError` + снятие префикса «Error:»). Вне UI-скоупа остался только серверный параметр поиска для `/api/audit` (Rust) — в UI поиск честно помечен «по загруженным строкам».

## Общая оценка

Основа UI крепкая: продуманная токен-система в `globals.css`, аккуратно тюнингованная тёмная тема, единый паттерн empty-state, цельная типографика (Nunito/Manrope/JetBrains Mono). Системные проблемы четыре:

1. **Светлая тема не дотюнена по контрасту** — тёмная получила GitHub-палитру, светлая осталась на сырых Tailwind-500.
2. **Опасные действия срабатывают без подтверждения**, а одна кнопка подтверждения подписана неправильно.
3. **Мобильная версия имеет структурные дефекты** — от двойной шапки до недоступных табов.
4. **Дизайн-система «протекает»** — страницы обходят собственные примитивы (Badge, Field, ConfirmDialog, chart-токены).

## Этап 1 — Контраст и читаемость (критично)

Светлая тема системно проваливает WCAG AA (4.5:1 для текста). Тёмная в порядке — правки только светлой.

- [ ] **`--success`/`--warning` светлой темы нечитаемы как текст** (1.86–2.28:1, ~84 использования в 29 файлах).
  Файлы: `ui/src/app/globals.css:96-97`, `ui/src/components/ui/button.tsx:14-18`, `ui/src/components/ui/badge.tsx`, `ui/src/components/ui/error-banner.tsx`.
  Фикс: затемнить только светлые значения (`--success ≈ #15803d`, `--warning ≈ #a16207`, проверить ≥4.5:1 на `--card #eaeff7`); добавить токены `--success-foreground`/`--warning-foreground` и заменить ими хардкод `text-white` в button/badge. Убрать костыль `text-amber-600 dark:text-amber-400` в `monitor/page.tsx` (там уже заметили проблему) и несуществующий класс `text-success-foreground` в `access/page.tsx:318`.

- [ ] **«Третичный» текст через `text-muted-foreground/30–70`** — ~156 мест в 41 файле, 9 разных ступеней opacity, контраст 1.75–2.9:1. Затрагивает подсказки всех форм (`ui/src/components/ui/field.tsx:45` — /60), композер чата, подписи метрик монитора.
  Фикс: один токен `--muted-foreground-subtle` (light/dark, ≥4.5:1 на `--card`), экспорт через `@theme inline`; заменить /50–/70 на осмысленном тексте, opacity ≤/40 оставить только декоративным иконкам.

- [ ] **Белый текст на primary-кнопке**: тёмная тема 2.64:1 (белый на #6b9eff), светлая 3.74:1.
  Фикс: в `.dark` задать `--primary-foreground: #1c2130` (совпадает с фоном, 6.1:1 — стандартный ход при осветлённом primary); в светлой либо слегка затемнить `--primary` (#3b6ef5–#2f62e8), либо принять 3.7:1 и поднять кегль/вес.

- [ ] **Цвета уровней логов в светлой теме**: WARN ~1.5:1, INFO ~2.1:1 (`monitor/page.tsx`, LEVEL_COLORS).
  Фикс: убрать /60–/70 из LEVEL_COLORS (полные токены: INFO `text-primary`, WARN `text-warning`, DEBUG `text-muted-foreground`) — сработает вместе с затемнением `--warning` из первого пункта.

- [ ] **Плейсхолдер — единственный лейбл ключевых инпутов при ~1.5–2:1** (композер чата, токен на логине).
  Фикс: `aria-label` из существующих ключей + поднять плейсхолдеры до дефолтного `placeholder:text-muted-foreground` из `input.tsx:11`, убрав переопределения /30–/50.

## Этап 3 — Мобильная версия

- [ ] **Двойная шапка на /chat/ и /workspace/** — в `ui/src/app/(authenticated)/layout.tsx:142` сравнение `pathname !== "/chat"` не учитывает trailing slash (`trailingSlash: true` в `next.config.ts`). Подтверждено скриншотом: две шапки, два гамбургера.
  Фикс: нормализовать `pathname.replace(/\/+$/, "")`; лучше — общий список «страниц со своей шапкой» одной константой для layout и sidebar.

- [ ] **`maximumScale: 1` блокирует pinch-zoom** (`ui/src/app/layout.tsx:28`, WCAG 1.4.4) — при 10px-моно, JSON и логах это боль. Убрать (soft-keyboard уже обрабатывает `interactiveWidget`).

- [ ] **Аудит: строки фиксированной ширины без переносов** — вся лента становится горизонтальным скролл-трапом на телефоне (`monitor/page.tsx`).
  Фикс: как у логов — `flex-col gap-1 md:flex-row`, время+агент+бейдж одной оборачиваемой строкой, убрать фиксированные `w-20` ниже md.

- [ ] **Тач-цели 24–28px** на destructive/primary действиях строк (approve/reject в access, delete в ProviderRow, сессии чата) при собственной базе h-9.
  Фикс: паттерн `h-9 md:h-7` / `size-9 md:size-6`; убрать `scale-75` с мобильного свитча автоскролла.

- [ ] **Скрытые табы без аффорданса скролла** — 8 табов монитора не влезают на телефоне, скроллбар убран `scrollbar-none`, «Одобрения»/«Куратор» не обнаружить. Тот же паттерн в providers и AgentEditDialog.
  Фикс: извлечь общий `ScrollableTabsList` с fade-градиентом у края (или `scrollbar-thin`); ниже md — свернуть табы монитора в Select, привязанный к `?tab=`.

- [ ] **Ниже lg чат теряет весь ContextBar** — модель, токены, предупреждение о переполнении контекста, ошибки стрима, доступ к чекпоинтам.
  Фикс: компактный вариант в мобильную панель (бейдж модели + 4px прогресс-бар токенов + триггер чекпоинтов; CheckpointPanel уже Sheet и работает на малых экранах).

- [ ] **Hover-only элементы мертвы на тач и невидимы для клавиатуры**: таймстемпы сообщений (`MessageItem.tsx` — opacity-0 без md:-гарда), действия над сообщениями и сессиями (`MessageActions.tsx:380`, `chat/page.tsx:705`).
  Фикс: таймстемпам `md:opacity-0 md:group-hover:opacity-100` (всегда видимы на <md); действиям добавить `md:group-focus-within:opacity-100` (паттерн уже есть в `memory/page.tsx:270`).

- [ ] **Портал `#mobile-page-actions` — мёртвый код** (ни одна страница ничего не инжектит, в шапке висит пустой разделитель). Либо подключить через PageHeader + `createPortal`, либо удалить.

## Приложение: мелкие находки (low, проверены только автором ревью)

**Токены/примитивы:** типографика заголовков модалок неконсистентна и равна page-title; reasoning-markdown без `dark:prose-invert`.
**Страницы:** иконка удаления агента — повёрнутый LogOut, и все кнопки удаления крутятся при любом сохранении; скелетон providers показывает грид карточек, а страница грузится табами; Routing Rules спрятаны в табе «Каналы»; «Session: » хардкодом; бейджи 9/10/12px вперемешку; три анатомии create-диалогов; blank-страницы на legacy-редиректах (/statistics и др.); '...' вместо Skeleton в watchdog; нет счётчика pending на табе «Одобрения»; хардкод-тост в setup; шаги wizard — только иконки без названий.
**Чат:** хоткей «Escape — остановить генерацию» задокументирован, но не работает; message-grouping CSS в globals.css — мёртвый код при виртуализации; дропдаун модели читается как статичный текст; фейковая всегда-зелёная точка «онлайн» на welcome-экране.
**Workspace:** постоянная error-подсказка под каждым PDF; «Открыть обозреватель» читается как браузер; emoji 🖼 вместо иконок; хардкод-английский aria у меню строк; Sheet без accessible title; контент редактора не выровнен с тулбаром.
**A11y:** 5px-скроллбар с 20%-thumb почти невидим (Firefox вообще без стилей); `role="listitem"` без `role="list"`; toggle токена всегда анонсирует «Show token»; error-баннеры логина/setup не анонсируются.
