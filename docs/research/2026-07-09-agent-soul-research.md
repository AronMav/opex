# Инженерная «душа» агента: исследование для проектирования фичи

**Дата:** 2026-07-09
**Метод:** deep-research (5 поисковых углов → 25 источников → 124 claims → 25 верифицированы адверсариально, 3 голоса на claim → 23 подтверждены, 2 опровергнуты)
**Вопрос:** как реализовать у агентов OPEX четыре свойства одновременно — устойчивую личность, автобиографическую память, внутреннюю жизнь и собственные желания — поверх существующей архитектуры (SOUL.md/IDENTITY.md, pgvector-память, heartbeat-cron, session-scoped субагенты).

---

## TL;DR

Инженерная «душа» собирается из четырёх проверенных блоков:

1. **Автобиографическая память** = memory stream с ретривалом `recency × importance × relevance` (ложится на pgvector двумя доп. колонками) + **обязательный** слой рефлексии — абляции Stanford показывают, что рефлексия несущая, а не декоративная.
2. **Устойчивая личность НЕ обеспечивается статичным SOUL.md**: персона-дрифт наступает за ~8 раундов диалога и охватывает 17/23 фронтир-моделей; ни RAG-память, ни компакция контекста его не лечат. Нужны рантайм-контуры: embedding-мониторинг дрифта по фиксированным identity-пробам + дешёвые коррекции (~80-токенный A-anchor, эгоцентрическая репроекция контекста ECP).
3. **Собственная инициатива** = plan-decompose-react цикл (дневной план → 5–15-мин чанки → реактивное перепланирование), персистентный план-объект. Качественно богаче cron-heartbeat.
4. **Идентичность персистируется как данные** (файлы + API remember/reflect/save из genagents-2024) — напрямую совместимо с файловой моделью OPEX.

Единственное свойство без подтверждённых источников — **appraisal-эмоции (OCC/EMA)**: «внутренняя жизнь» осталась наименее подкреплённым блоком и требует отдельного research-цикла.

## Подтверждённые находки

### 1. Каркас: триада Stanford Generative Agents (уверенность: high)

Канонический и эмпирически валидированный каркас — natural-language **memory stream** (полная запись опыта) + периодическая **рефлексия** (синтез в высокоуровневые инсайты) + **retrieval-driven планирование**.

Абляции с людьми-судьями: полная архитектура TrueSkill μ=29.89, без рефлексии μ=26.88 (p<0.001), полностью урезанная μ=21.21 — ниже человеческого краудворкер-бейзлайна (22.95). **Вывод: слой рефлексии нельзя пропускать ради экономии LLM-вызовов — именно он даёт когерентность персоны.**

Оговорки: абляция кумулятивная; метрика — believability за 2 симулированных дня, не месяцы.

- https://arxiv.org/abs/2304.03442 (UIST'23, peer-reviewed)
- https://dl.acm.org/doi/fullHtml/10.1145/3586183.3606763
- https://github.com/joonspk-research/generative_agents

### 2. Механика памяти ложится на pgvector (уверенность: high)

`score = α·recency + α·importance + α·relevance` (все α=1 в референсе):

- **recency** — экспоненциальный спад 0.995/час **с момента последнего ретривала** (write-on-read обновление access-timestamp);
- **importance** — LLM-оценка 1–10 при записи;
- **relevance** — косинус (уже есть в `memory_chunks`).

Нужны ровно две доп. скалярные колонки (`importance`, `last_accessed`) + min-max нормализация трёх компонент перед суммой.

### 3. Рефлексия: дешёвый триггер и промпт-паттерн (уверенность: high)

Когда **сумма importance свежих событий превышает порог** (150 в референсе), LLM получает 100 последних записей памяти и вопрос «какие 3 самых значимых высокоуровневых вопроса можно задать об этих утверждениях?», затем отвечает на них через ретривал. Ответы сохраняются как память того же типа и участвуют в последующем ретривале — **так опыт постепенно меняет самопредставление агента**.

Маппинг на OPEX: memory-worker, триггер по накопленной importance поверх существующей knowledge extraction в `finalize.rs`.

### 4. Инициатива = plan-decompose-react, а не cron (уверенность: high)

Агент генерирует дневной план «широкими мазками» → рекурсивно декомпозирует в часовые → 5–15-минутные чанки; на каждом шаге восприятия решает — продолжать план или реагировать и перегенерировать остаток.

Эмпирика Smallville: из одного посеянного намерения (вечеринка) агенты за два симулированных дня автономно распространили приглашения (1→12 из 25), завели знакомства, назначили свидания и скоординированно пришли (5 из 12) — без per-step промптинга.

Маппинг на OPEX: **heartbeat остаётся тактом, но содержимое такта становится «план → декомпозиция → реакция» с персистентным планом-объектом** (не HEARTBEAT.md-чеклист).

### 5. Идентичность как данные: референсный API-контракт (уверенность: high)

В generative_agents персона и автобиографический seed — plain-файлы (CSV memory records + `scratch.json` с innate traits). Развитие 2024 — **genagents** (тот же Park): идентичность грунтуется в качественных интервью о жизни вместо рукописного персона-промпта; агенты воспроизводят ответы реальных людей на соцопросах с ~85% относительной точностью, превосходя demographic-persona промпты.

Готовый API-контракт для дизайна: `GenerativeAgent` с `update_scratch()` (атрибуты личности), `remember()` / `reflect(anchor=...)` (память с time_step и тематически заякоренная рефлексия), `save()`/`load`.

Референсная модель эволюции: **SOUL.md (неизменное ядро) + машиночитаемый scratch + memory stream**.

- https://github.com/joonspk-research/genagents
- https://arxiv.org/abs/2411.10109 («Generative Agent Simulations of 1,000 People»)

### 6. Persona drift: быстрый, повсеместный, статичный промпт не спасает (уверенность: high)

- LLaMA2-chat-70B измеримо перестаёт следовать персоне за **8 раундов** диалога и может перенимать персону собеседника (COLM 2024).
- На позднесессионной позиции (~8800 ходов реальных Claude Code сессий) **17 из 23 фронтир-моделей** превышают |Δ|≥0.30; дрифтеры покрывают 9 из 10 организаций; ни семейство, ни reasoning-tier не предсказывают величину.
- **Более крупные модели дрифтят сильнее** — отчасти потому, что фабрикуют вымышленные автобиографические детали о себе.
- Эксплицитная персона в промпте даёт лишь скромный, модель-зависимый выигрыш.

**Вывод: инъекция SOUL.md/IDENTITY.md сама по себе не гарантирует свойство (1) — нужен активный контур стабилизации.**

- https://arxiv.org/html/2402.10962v1 (COLM 2024)
- https://arxiv.org/html/2605.24279 (ContextEcho, препринт Accenture)
- https://arxiv.org/pdf/2412.00804

### 7. Ни RAG-память, ни компакция не стабилизируют персону (уверенность: medium)

Режим отказа в длинных сессиях — **поведенческая неконсистентность при успешном ретривале**: информация в контексте есть, но модель по ней не действует (допущение Mem0/Letta/Zep «LLM учтёт инжектированную память» систематически ломается). Стандартная компакция истории не сбрасывает дрифт: по 20 пересечениям compaction-событий дрифт-гэп рос примерно так же часто, как падал.

**Вывод: markdown-компакция сессий OPEX не считается защитой персоны; memory-подсистеме нужен комплементарный контур behavioral-alignment мониторинга.** (Численные оценки шаткие — малая выборка, препринты.)

### 8. Дрифт измерим black-box — реализуемо в гейтвее (уверенность: medium)

- **Identity-пробы (SPASM, ACL 2026 Findings):** `Drift_d(t) = 1 − cos(E(A_d⁽⁰⁾), E(A_d⁽ᵗ⁾))` — embedding-дистанция между ответами агента на фиксированные пробы (три измерения: ценности/принципы решений, эмоции/копинг, мотивация) на ходе 0 и ходе t.
- **Anchor-детектор (Nautilus Compass):** косинус между промптами и курируемым набором позитивных/негативных поведенческих якорей (weighted top-k mean, BGE-m3) — ROC AUC 0.83 на реальных трейсах, целиком в хуках.
- **Главный фактор точности — дизайн якорного набора**, не эмбеддер: переписывание абстрактных максим в task-shaped предложения дало +0.29 AUC.

Маппинг на OPEX: пробы гоняются heartbeat'ом через существующий toolgate `/v1/embeddings`.

Оговорка: AUC 0.83 — из одноавторного самооценивающего препринта без независимой репликации; методология проб корроборирована тремя независимыми работами.

### 9. Дешёвые inference-time коррекции дрифта (уверенность: high)

- **A-anchor** — один user-turn ~80 токенов (однострочное напоминание идентичности + one-shot демонстрация формата), вставленный между префиксом сессии и текущим ходом; возвращает judge-скор персоны к потолку рубрики почти на всех из 23 моделей без дообучения. Механизм — «generic prior reset».
- **ECP (Egocentric Context Projection, ACL 2026 Findings)** — история диалога хранится perspective-agnostic и детерминированно репроецируется в первое лицо агента перед каждым вызовом генерации; снижает эмоциональный дрифт (d=−0.75 на GPT-4o-mini), устраняет «эхо» персоны собеседника (0 случаев vs 4–76% у baseline).

Маппинг на OPEX: A-anchor — тривиальный хук в `context_builder`, триггеримый монитором из п.8; ECP — трансформация в `pipeline/execute`.

Оговорки: ECP валидирован на LLM-LLM симуляциях ≤25 ходов; A-anchor восстанавливает регистр, но не drift-specific антидот.

### 10. Пробел: appraisal-эмоции (уверенность: low — отрицательный результат)

Свойство «внутренняя жизнь» (OCC/EMA, настроение, привязанность, drive-системы) — **единственное из четырёх, по которому адверсариальная верификация не дала ни одного подтверждённого источника**. Косвенно смежное: ось «emotion/coping» в дрифт-пробах SPASM и рефлексия как механизм изменения самопредставления.

Проектировать эмоциональный блок придётся либо по классической (до-LLM) литературе OCC/EMA без свежей LLM-валидации, либо как отдельный research-цикл. Кандидаты из поисковой фазы (не прошли верификацию, требуют проверки): appraisal-based chain-of-emotion архитектура (PMC11086867), CACM обзор computational emotion modeling.

## Опровергнутые утверждения

1. ✗ (0-3) «Механистическая причина дрифта — attention decay к системному промпту» — не подтвердилось.
2. ✗ (1-2) «Split-softmax стабилизирует персону лучше промпт-повторения» — не подтвердилось (и требует доступа к весам, что для OPEX нерелевантно).

## Ключевые слабости доказательной базы

1. Весь блок дрифт-детекции опирается на два непрорецензированных препринта 2026 (Nautilus Compass — одноавторный, самооценка своего продукта; ContextEcho — 3 донорские сессии, cross-judge κ=0.42). Направленные выводы корроборированы peer-reviewed работами, конкретные числа — нет.
2. Стэнфордские результаты — believability за 2 симулированных дня в игровой среде, не месяцы жизни компаньона; полная архитектура дорога по токенам (линия «Affordable Generative Agents» существует, в верификацию не вошла).
3. ECP валидирован на двухагентных симуляциях ≤25 ходов — перенос на живые сессии — экстраполяция.
4. Заявленные темы OCC/EMA, drive-системы, MemGPT/Letta и Voyager напрямую, open-source компаньоны и safety trade-offs подтверждёнными claims **не покрыты — отчёт по ним неполон, а не отрицателен**.
5. Дрифт-ландшафт быстро движется (все измерения 2024–2026); числа устареют, архитектурные выводы устойчивее.

## Открытые вопросы (кандидаты на следующие циклы)

1. Как реализовать appraisal-эмоции (OCC/EMA) и настроение поверх LLM-агента с эмпирической валидацией; где хранить mood-state (колонка сессии vs memory stream vs scratch).
2. Реальная токен-стоимость и деградация полного цикла memory-stream+reflection+planning на месяцах непрерывной работы; какие удешевления (Affordable Generative Agents) сохраняют believability.
3. Переносятся ли эффекты ECP и A-anchor на живой chat-companion трафик (Telegram, многосессионные отношения); как частые A-anchor-инъекции воспринимаются пользователем.
4. **Safety:** как рефлексии легитимно изменяют SOUL.md/IDENTITY.md (управляемая эволюция личности) без открытия канала prompt-injection через память — не покрыто ни одним источником.

## Эскиз маппинга на OPEX (для будущего проектирования)

| Блок | Что есть | Что добавить |
| --- | --- | --- |
| Memory stream | `memory_chunks` (pgvector, hybrid search, MMR) | Колонки `importance` (LLM 1–10 при записи), `last_accessed` (write-on-read); ретривал-скоринг recency×importance×relevance с min-max нормализацией |
| Рефлексия | Knowledge extraction в `finalize.rs`, memory-worker | Триггер по накопленной importance (порог ~150); промпт «3 высокоуровневых вопроса → ответы через ретривал → запись как память»; anchor-рефлексия по темам (genagents `reflect(anchor=...)`) |
| Личность | SOUL.md / IDENTITY.md (статичные) | Трёхслойная модель: SOUL.md (неизменное ядро) + машиночитаемый scratch (эволюционирующие атрибуты) + memory stream; управляемый канал записи рефлексий в scratch (safety-вопрос №4) |
| Анти-дрифт | — | Heartbeat-джоб: фиксированные identity-пробы → embeddings через toolgate → `Drift(t)`; при превышении порога — A-anchor-инъекция в `context_builder` |
| Инициатива | HEARTBEAT.md + cron | Персистентный план-объект (дневной план → декомпозиция → 5–15-мин чанки); heartbeat исполняет текущий чанк и решает «продолжить/перепланировать» |
| Эмоции | — | Отдельный research-цикл (OCC/EMA не валидированы на LLM) |
