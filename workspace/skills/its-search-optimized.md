# Методика оптимального поиска по ИТС (its.1c.ru)

## Общая стратегия

Поиск ответа на вопрос по 1С в документации ИТС — многоэтапный процесс. Прямой поиск по одному запросу редко даёт точный ответ. Необходимо комбинировать инструменты и перебирать базы документации.

## Этап 1. Определение целевых баз документации

Базы документации ИТС упорядочены по версиям платформы. Приоритет перебора:

1. **v854doc** — платформа 8.5.4 (тестовая, самая свежая)
2. **v851doc** — платформа 8.5.1
3. **v8327doc** — платформа 8.3.27
4. **v8326doc** — платформа 8.3.26
5. **v8322doc** — платформа 8.3.22 (если актуально)
6. **v8323doc** — платформа 8.3.23 (если актуально)

Если пользователь явно указал версию платформы — начинать с соответствующей базы.

Дополнительные базы (для специфических тем):
- **metod8dev** — методические материалы для разработчиков
- **freshex1** — документация по 1С:Фреш
- **pub** — публикации и статьи

## Этап 2. Первичный поиск инструментом `its`

Выполнить `its(action="search", query="<ключевые слова>")` **без указания db** — поиск по всем базам сразу.

Если результатов нет или они нерелевантны — повторить с уточнённым запросом (убрать стоп-слова, оставить только ключевые термины).

## Этап 3. Фильтрация по базе документации

**Важно:** параметр `db` в запросе сейчас игнорируется сервером (поиск ИТС всегда идёт по объединённой базе morphmerged). НЕ перебирать базы повторными запросами с `db=` — это одинаковая выдача и впустую потраченные 5–20 секунд на каждый вызов.

Вместо этого:
1. Каждый результат поиска содержит поле `db` (например `v854doc`, `answers1c`) — **фильтровать выдачу по этому полю** согласно приоритету баз из Этапа 1.
2. Уточнять сам запрос терминами версии («8.3.27», «регламентные задания сервер») — это меняет выдачу, в отличие от `db=`.

**Правило:** не более 3-х переформулировок запроса на один вопрос без чтения результатов.

## Этап 4. Чтение найденных фрагментов

Для каждого релевантного результата выполнить `its(action="read", ref="<ref из поиска>")`.

**Важно:** при чтении больших глав (например, Глава 10) контент может быть обрезан. В этом случае:
- Выполнить `its(action="search", query="<уточнённый запрос>")` для поиска конкретного подраздела внутри главы и читать его ref напрямую
- `web_fetch(url="https://its.1c.ru/<ref>")` работает ТОЛЬКО для публичных страниц — подписочный контент отдаёт анониму 403 Forbidden (залогиненная сессия есть только у инструмента `its`)

## Этап 5. Расширенный поиск через search_web

Если инструмент `its` не дал результата — использовать `search_web`:

```
search_web(query="<ключевые слова> site:its.1c.ru")
```

Это позволяет найти страницы, которые могли быть пропущены внутренним поиском ИТС.

Для найденных URL — читать через `web_fetch`.

## Этап 6. Поиск в смежных источниках

Если в основных базах документации ответ не найден:

1. **Методические материалы:** `its(action="search", query="<запрос> методика")` и отобрать результаты с `db=metod8dev` из выдачи
2. **Партнёрские конференции / статьи:** `search_web(query="<запрос> site:its.1c.ru")`
3. **v8.1c.ru:** `search_web(query="<запрос> site:v8.1c.ru")` — раздел вопросов по лицензированию
4. **Инфостарт / форумы:** `search_web(query="<запрос> 1С лицензирование")` — для неофициальных, но практических ответов

## Этап 7. Верификация и цитирование

При нахождении ответа:

1. **Точно указать источник:** название главы, раздела, базу документации
2. **Привести точную цитату** из документации
3. **Указать ссылку** на страницу ИТС
4. **Если точной цитаты нет** — честно сообщить: «Прямого утверждения в документации не обнаружено. Однако следующие фрагменты в совокупности подтверждают логику: ...»

## Антипаттерны (чего НЕ делать)

- Не ограничиваться одним поисковым запросом
- Не выдавать результат поиска по Фреш (freshex1) за общую документацию платформы
- Не утверждать наличие цитаты, если она не найдена дословно
- Не игнорировать версию платформы, указанную пользователем
- Не делать более 3-х поисковых запросов подряд без чтения результатов
