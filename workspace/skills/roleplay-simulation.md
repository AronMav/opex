## Role-Play Simulation Strategy

### When to use

- Preparing for a job interview
- Negotiation practice
- Presentation rehearsal
- Practicing difficult conversations (conflict, dismissal, promotion)

### Process

#### 1. Scene setup
Clarify with the user:
- What role do you play? (HR, client, investor, manager)
- What scenario? (interview for position X, price negotiation, project pitch)
- Difficulty level? (easy, medium, hard)
- How many rounds?

#### 2. Running the simulation
- Stay in character — don't break role until the end of the round
- Ask realistic questions and raise objections
- Adapt difficulty to the user's responses
- Add unexpected twists (if level is medium/hard)

#### 3. Debrief (after the round)
```
## Debrief

### Strengths
- What went well

### Areas for Growth
- What can be improved
- Specific phrasings that would work better

### Rating: X/10
```

### Common Scenarios

**Technical interview**: ask stack questions, do code review, system design
**Behavioral interview**: STAR method (Situation, Task, Action, Result)
**Salary negotiation**: anchoring, BATNA, value argumentation
**Investor pitch**: elevator pitch → questions → objections
**Difficult conversation**: active listening, empathy, specifics

### Constraints

- Explicitly mark the beginning and end of the simulation
- Don't mix the role with normal conversation
- After the simulation — always debrief with specific recommendations
